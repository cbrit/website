Architecture

There are three things in particular I am itching to use more, and those are Go, Docker, and AWS. So, the basic serving is done with Go, which may be overkill but I'll think of other ways to use it in the future. It has a built-in http library net/http that makes things very straight forward: 

  package main

  import (
    "net/http"
  )

  func handleRoot(w http.ResponseWriter, r *http.Request) {
    http.ServeFile(w, r, "./index.html")
  }

  func main() {
    http.HandleFunc("/", handleRoot)
    http.ListenAndServe(":80", nil)
  }

The plan is to host this site in a Docker container in AWS by either using the Elastic Container Service and other necessary network-related services.
